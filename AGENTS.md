# Sokki — Agent Guide

Sokki is a local-first macOS menu bar dictation app (Korean / English /
code-mixed speech). This file is the working contract for AI agents making
changes in this repo. For installing the app itself, follow
[INSTALL.md](INSTALL.md). For product behavior and troubleshooting, read
[docs/onboarding.md](docs/onboarding.md).

## Layout

- `Sources/SokkiCore/` — engine-agnostic product logic (settings, dictation pipeline, SQLite history, vocabulary, insertion). No AppKit. Everything here must stay testable without a GUI session.
- `Sources/Sokki/` — the app: recording state machine, Apple Speech streaming, WhisperKit engine, Carbon hotkey + fn CGEventTap, HUD panel, SwiftUI settings/onboarding.
- `scripts/` — the only supported build/test/package entry points. Use them instead of raw `swift build` invocations so behavior matches CI.
- `.hoyeon/` and `dist/` are untracked; never commit them.

## Build, Test, Verify

```sh
./scripts/build.sh              # SwiftPM build (Command Line Tools is enough)
./scripts/test.sh               # SokkiCore test suite — must stay green
./scripts/package-app.sh        # dist/Sokki.app (icon + ad-hoc signing; APP_VERSION env optional)
./scripts/desktop-qa-smoke.sh   # headless smoke: menu bar/settings/overlay wiring
./scripts/make-dmg.sh           # dist/Sokki-<version>.dmg from the packaged app
```

Before opening a PR: build, tests, and the smoke script must pass. For any
change touching recording, speech, permissions, insertion, or the HUD, also
verify on-device: install to `/Applications`, trigger a real dictation, and
confirm the transcript lands (history table in
`~/Library/Application Support/Sokki/Sokki.sqlite` is the ground truth).

## Hard-Won Rules (violating these caused real bugs)

1. **Audio/speech callbacks must be `@Sendable`.** `SpeechSession` and friends are `@MainActor`; closure literals passed to `installTap`, `recognitionTask`, or CGEventTap callbacks inherit that isolation and crash with `dispatch_assert_queue` on the realtime thread unless marked `@Sendable`. Hop back to the main actor explicitly (`DispatchQueue.main.async` + `MainActor.assumeIsolated`) and move non-Sendable values across with the `UncheckedSendable` box pattern already in the codebase.
2. **Never trust "the streaming session will deliver."** speechd's first session in a process can silently return nothing. The defenses (recognizer prewarm at launch, cancelling unresponsive tasks, file-based re-recognition fallback in `RecordingCoordinator.stopAndInsert`) must survive refactors.
3. **Ad-hoc re-signing resets TCC.** Every rebuild invalidates Microphone/Speech/Accessibility grants for the installed app. After reinstalling during verification, re-grant permissions; for Accessibility, a stale System Settings toggle is not enough — `tccutil reset Accessibility com.hoyeon.Sokki`, re-register via the app's prompt, then re-enable.
4. **New `AppSettings` fields need tolerant decoding.** Follow the `decodeIfPresent ... ?? defaults` pattern so old installs upgrade cleanly, and keep the "settings decode tolerates missing new fields" test passing.
5. **The HUD panel must never steal focus.** It is a non-activating panel; auto-paste depends on keyboard focus staying in the target app.

## Delivery

- Work on a branch; open a PR; squash-merge. Direct pushes to `main` are reserved for trivial metadata (e.g. LICENSE).
- Write commits and PRs as project work. No AI/agent/model attribution anywhere (no "Generated by", no `Co-authored-by:` bot lines).
- PR bodies lead with reviewable results and evidence (screenshots for UI, history-table output for dictation), not process diaries.

## Release

Releases are tag-driven and fully automated:

```sh
git tag v<MAJOR.MINOR.PATCH>
git push origin v<...>
```

`.github/workflows/release.yml` then runs the core tests, packages the app
with the version stamped from the tag, builds the DMG, and publishes a GitHub
Release with the asset. There is no manual upload step. Fix a broken release
by deleting the tag+release and re-tagging, not by editing assets in place.

Releases are ad-hoc signed for now — INSTALL.md documents the quarantine
workaround. Developer ID signing + notarization is on the roadmap; when it
lands, remove those workarounds from INSTALL.md and README.

## Visual Assets

- App icon is generated, not hand-drawn: edit `scripts/generate-app-icon.swift`, run it, and commit both `AppResources/AppIcon.icns` and the refreshed `docs/screenshots/first-product/app-icon.png` (256px preview) together.
- README hero lives at `docs/assets/hero.png`; screenshots under `docs/screenshots/first-product/` are referenced by README/PRs — keep paths stable.
